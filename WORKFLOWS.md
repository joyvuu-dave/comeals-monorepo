# Workflows

This is the hub for workflow improvement notes. Each workflow in the app gets either:

- A brief inline section here (early-stage ideas, not yet substantial enough for a dedicated doc), OR
- A pointer to its dedicated document (once thinking is substantial enough)

When a workflow's hub section grows substantial, extract it to a dedicated document and replace the section here with a one-line pointer.

## What these documents are

These are **workflow improvement documents** — living docs that capture the desired user workflow, current pain points, and ideas for improvements. They preserve the _thinking_ behind decisions, not just the decisions, so future-you (or future contributors) can pick up where we left off without re-deriving rationale.

They are NOT plans (no commitments, no timelines) and NOT specs (no full technical detail). They are reference material: when we start implementing a specific item, we come back here, find the relevant section, and use it as the starting point for an implementation plan.
